The Problem

Drop the Proxy. Keep the Security.

Traditional DynamoDB apps required a proxy server just to regulate per-user access. Fine-Grained Access Control moves that logic into IAM — where it belongs.

❌ Before — Proxy Architecture
📱 App ──▶ 🔀 Proxy ──▶ ⚡ DynamoDB

⚠ Extra hop: added latency, ops burden, and cost. The proxy does nothing but check user IDs.

✅ After — FGAC Direct Access
📱 App ──▶ 🔑 STS ──▶ ⚡ DynamoDB
                     IAM evaluates FGAC inline

✔ Zero proxy. IAM conditions enforce row & attribute restrictions at the DynamoDB API level.

Policy Types

Four Modes of Control

Choose the policy that matches your app's access model. Switch any time with a single Terraform variable.

↔️

Horizontal

Each user can only access their own rows. Uses dynamodb:LeadingKeys to match the Cognito identity.

Row-Level
↕️

Vertical

All rows are accessible, but only whitelisted attributes are returned. Hides sensitive fields from clients.

Attribute-Level
⊕

Combined

Both row-level and attribute-level restrictions in one policy. The most common pattern for production apps.

Default · Recommended
👁️

Read-Only

Like horizontal, but permits only GetItem, Query, and BatchGetItem. Zero write access.

Read-Only
Example Policies

Production-Ready IAM Conditions

// horizontal_access.json — Row-level: users own their items
{
  "Version": "2012-10-17",
  "Statement": [{
    "Sid": "HorizontalAccessOwnItems",
    "Effect": "Allow",
    "Action": [
      "dynamodb:GetItem", "dynamodb:Query",
      "dynamodb:PutItem", "dynamodb:UpdateItem"
    ],
    "Resource": "arn:aws:dynamodb:*:*:table/GameScores",
    "Condition": {
      "ForAllValues:StringEquals": {
        "dynamodb:LeadingKeys": [
          "${cognito-identity.amazonaws.com:sub}"
        ]
      }
    }
  }]
}
// vertical_access.json — Attribute-level: restrict visible columns
{
  "Version": "2012-10-17",
  "Statement": [{
    "Sid": "VerticalAccessAllowedAttributes",
    "Effect": "Allow",
    "Action": ["dynamodb:GetItem", "dynamodb:Query"],
    "Resource": "arn:aws:dynamodb:*:*:table/GameScores",
    "Condition": {
      "ForAllValues:StringEquals": {
        "dynamodb:Attributes": [
          "UserId", "GameTitle",
          "Score", "Wins", "Losses"
        ]
      },
      "StringEqualsIfExists": {
        "dynamodb:Select": "SPECIFIC_ATTRIBUTES"
      }
    }
  }]
}
// combined_access.json — Row + attribute restrictions together
{
  "Version": "2012-10-17",
  "Statement": [{
    "Sid": "CombinedHorizontalAndVerticalAccess",
    "Effect": "Allow",
    "Action": [
      "dynamodb:GetItem", "dynamodb:Query",
      "dynamodb:PutItem", "dynamodb:UpdateItem"
    ],
    "Resource": "arn:aws:dynamodb:*:*:table/GameScores",
    "Condition": {
      "ForAllValues:StringEquals": {
        "dynamodb:LeadingKeys": ["${cognito-identity.amazonaws.com:sub}"],
        "dynamodb:Attributes": [
          "UserId", "GameTitle", "Score", "Wins", "Losses"
        ]
      }
    }
  }]
}
# terraform/terraform.tfvars
aws_region               = "us-east-1"
project_name             = "my-game-app"
environment              = "prod"
table_name               = "GameScores"
billing_mode             = "PAY_PER_REQUEST"
cognito_identity_pool_id = "us-east-1:xxxx-xxxx"

# Switch between: horizontal | vertical | combined | read_only
access_policy_type       = "combined"

---
# Then just run:
$ terraform init && terraform apply

# Outputs:
dynamodb_table_name = "GameScores"
dynamodb_table_arn  = "arn:aws:dynamodb:us-east-1:1234:table/GameScores"
iam_role_arn        = "arn:aws:iam::1234:role/my-game-app-prod-fgac-role"
Quick Start

Up in 3 Commands

Clone, configure, deploy. The DynamoDB table and all IAM roles are ready in minutes.

1

Clone & configure

Copy example.tfvars → terraform.tfvars and add your Cognito Identity Pool ID.

2

Run terraform init

Downloads the AWS provider and initialises the backend. Takes ~15 seconds.

3

Run terraform apply

Creates 4 AWS resources: table, IAM role, policy, attachment. Output includes all ARNs.

4

Integrate in your app

Use the role ARN with AssumeRoleWithWebIdentity. See usage.md for JS + Python examples.
